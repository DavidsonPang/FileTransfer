# FileTransfer - P2P 文件传输工具

基于 WebRTC 的纯 P2P 文件传输工具，支持跨网络传输，无需中继服务器传输数据。

## ✨ 特性

- 🌐 **跨网络传输**: 支持不同局域网的机器互传
- 🔒 **端到端加密**: 数据直接点对点传输，不经过任何服务器
- 🚀 **NAT 穿透**: 自动尝试 UDP 打洞建立直连
- 🖥️ **跨平台**: 支持 Windows、Mac、Linux
- 📊 **实时进度**: 显示传输速度和进度条
- ✅ **完整性校验**: 自动计算和验证 SHA256 哈希值
- 📱 **二维码支持**: 可以通过扫码快速交换连接信息

## 🎯 工作原理

```
机器 A (发送端)                            机器 B (接收端)
    │                                          │
    │ 1. 生成 Offer (包含网络信息)              │
    │ ────────────────────────────────────────> │
    │                                          │ 2. 生成 Answer
    │ <──────────────────────────────────────── │
    │                                          │
    │ 3. WebRTC 建立 P2P 连接                  │
    │ <══════════════════════════════════════> │
    │                                          │
    │ 4. 直接传输文件 (加密)                    │
    │ ════════════════════════════════════════> │
    │                                          │
```

**关键点：**
- 使用免费的 STUN 服务器（Google）仅用于发现公网 IP
- 文件数据完全 P2P 传输，不经过任何第三方
- 手动交换 SDP 信息（通过微信、QQ 等），无需信令服务器

## 🚀 使用方法

### 发送文件

```bash
python sender.py <文件路径>
```

示例：
```bash
python sender.py movie.mp4
```

程序会：
1. 显示一段压缩后的连接信息
2. （可选）显示二维码
3. 等待你输入接收端返回的信息

### 接收文件

```bash
python receiver.py
```

或指定保存目录：
```bash
python receiver.py --output /path/to/save
```

程序会：
1. 等待你输入发送端的连接信息
2. 生成响应信息，需要发送给发送端
3. 自动开始接收文件

## 🔧 高级用法

### 传输大文件

程序已优化处理大文件（10GB+）：
- 自动分块传输（64KB 每块）
- 流控机制防止内存溢出
- SHA256 完整性校验

### 网络要求

✅ **大多数情况都能成功** (80-90%)：
- 家庭宽带 ✓
- 公司网络 ✓
- 校园网 ✓
- 4G/5G 热点 ✓

❌ **可能失败的情况**：
- 严格的对称型 NAT（少见）
- 企业防火墙完全阻止 UDP

### 失败了怎么办？

如果 NAT 穿透失败，程序会提示：

```
✗ 连接失败，可能是 NAT 穿透失败
建议使用 ZeroTier 或 Tailscale 创建虚拟局域网
```

**解决方案：**

1. **使用 VPN 组网**（推荐）
   - [ZeroTier](https://www.zerotier.com/): 免费，简单易用
   - [Tailscale](https://tailscale.com/): 基于 WireGuard，速度更快

2. **端口转发**（如果有公网 IP）
   - 在路由器上配置端口转发
   - 然后使用手动 IP 连接

## 🔐 安全性

- ✅ **端到端加密**: WebRTC 默认使用 DTLS/SRTP 加密
- ✅ **完整性校验**: SHA256 哈希验证文件未被篡改
- ✅ **无第三方存储**: 文件数据不经过任何服务器
- ✅ **手动信令**: 交换信息需要人工确认，防止中间人攻击

## ❓ 常见问题

**Q: 为什么不能直接输入对方 IP？**
A: 因为对方在 NAT 后面，你看到的是路由器 IP，无法直达。WebRTC 通过 STUN/ICE 自动找到正确的路径。

**Q: STUN 服务器会看到我的文件吗？**
A: 不会！STUN 只用于发现你的公网 IP 地址，不传输任何文件数据。

**Q: 速度能达到多少？**
A: 取决于双方的上行带宽。通常能达到理论上限的 80-90%。

**Q: 支持断点续传吗？**
A: 当前版本不支持。建议传输前确保网络稳定，或先打包压缩大文件。

**Q: 可以同时传输多个文件吗？**
A: 可以先打包成 zip/tar 文件再传输。
