# 手动测试指南

## 环境准备

### 1. 安装依赖

```bash
pip install cryptography>=42.0.0
pip install aiortc>=1.6.0
pip install tqdm>=4.66.0
pip install qrcode>=7.4.2
pip install Pillow>=10.0.0
```

**注意：** `aiortc` 需要编译环境：
- **Windows**: 安装 [Visual Studio Build Tools](https://visualstudio.microsoft.com/downloads/)
- **Mac**: 运行 `xcode-select --install`
- **Linux**: `sudo apt-get install python3-dev libavformat-dev libavdevice-dev libavcodec-dev libswscale-dev`

### 2. 创建测试文件

```bash
# 创建一个小测试文件
echo "Hello FileTransfer Test!" > test_small.txt

# 创建一个大一点的测试文件 (10MB)

```

## 测试场景

### 场景 1: 同一台机器测试（验证基础功能）

1. **打开第一个终端 - 接收端**
```bash
cd FileTransfer
python receiver.py
```

你会看到类似的输出：
```
============================================================
等待发送端的连接信息...
============================================================

请输入发送端提供的信息:
```

2. **打开第二个终端 - 发送端**
```bash
cd FileTransfer
python sender.py test_small.txt
```

你会看到类似的输出：
```
正在收集网络信息...
✓ 网络信息收集完成

============================================================
请将以下信息发送给接收端（可以通过微信、QQ 等方式）
============================================================
H4sIAAAAAAAA/6VWTY/bNhC9+1cQvgiQZVmyP...（一长串字符）
============================================================

请输入接收端返回的信息:
```

3. **复制信息到接收端**
   - 复制发送端显示的长字符串
   - 粘贴到接收端的输入提示处
   - 按回车

4. **接收端会生成响应**
```
============================================================
请将以下信息发送给发送端
============================================================
H4sIAAAAAAAA/5VWTY8bNxC8+1cM...（另一串字符）
============================================================

✓ 等待连接建立...
```

5. **复制响应到发送端**
   - 复制接收端生成的字符串
   - 粘贴到发送端的输入提示处
   - 按回车

6. **观察传输过程**

发送端应该显示：
```
✓ 已接收对方响应，等待连接建立...
连接状态: connecting
连接状态: connected
✓ 连接已建立，开始传输...

文件名: test_small.txt
大小: 24.00 B
SHA256: a3b5c8d1e2f3...

发送进度: 24B/24B [████████████████████] 100% 1.2KB/s
✓ 文件传输完成！
```

接收端应该显示：
```
✓ 数据通道已建立

文件名: test_small.txt
大小: 24.00 B
SHA256: a3b5c8d1e2f3...

接收进度: 24B/24B [████████████████████] 100%

正在校验文件...
✓ 文件接收完成且校验通过！
✓ 文件已保存到: ./received_test_small.txt
```

### 场景 2: 不同机器测试（实际使用场景）

#### 前提条件
- 两台机器都能访问互联网
- 两台机器不需要在同一局域网

#### 步骤
1. **机器 A**（例如家里的 Windows 电脑）运行接收端
2. **机器 B**（例如公司的 Mac）运行发送端
3. 通过微信/QQ/邮件交换连接信息
4. 观察 NAT 穿透和传输过程

#### 预期结果
- 如果 NAT 穿透成功：连接状态显示 `connected`，文件直接 P2P 传输
- 如果 NAT 穿透失败：连接状态显示 `failed`，程序会提示使用 VPN 方案

### 场景 3: 大文件传输测试

```bash
# 创建 1GB 测试文件
dd if=/dev/zero of=test_1gb.bin bs=1M count=1024

# 发送
python sender.py test_1gb.bin
```

观察：
- 内存使用是否稳定（不应该持续增长）
- 传输速度是否合理（取决于带宽）
- SHA256 校验是否通过

## 常见问题排查

### 问题 1: 导入 aiortc 失败

```python
python -c "import aiortc; print('OK')"
```

如果失败，重新安装：
```bash
pip uninstall aiortc
pip install aiortc --no-cache-dir
```

### 问题 2: 连接状态一直是 connecting

可能原因：
- 防火墙阻止 UDP（检查防火墙设置）
- 网络环境是严格对称型 NAT（使用 VPN 方案）
- STUN 服务器无法访问（测试：`ping stun.l.google.com`）

### 问题 3: 校验失败

可能原因：
- 网络不稳定导致数据丢失
- 传输过程被中断
- 解决：重新传输

### 问题 4: Windows 上编译 aiortc 失败

下载预编译版本：
```bash
pip install https://github.com/aiortc/aiortc/releases/download/1.6.0/aiortc-1.6.0-cp39-cp39-win_amd64.whl
```
（根据你的 Python 版本选择对应的 wheel 文件）

## 性能基准

### 预期性能（参考）
- **局域网**: 50-100 MB/s（取决于网卡速度）
- **跨网络**: 取决于上行带宽，通常能达到理论值的 80-90%
- **CPU 使用**: 单核 10-30%
- **内存使用**: 50-100MB（稳定，不随文件大小增长）

### 成功率基准
- **家庭宽带 ↔ 家庭宽带**: 90%+
- **家庭宽带 ↔ 4G/5G**: 85%+
- **企业网络 ↔ 家庭宽带**: 70%+
- **企业网络 ↔ 企业网络**: 50%+（取决于防火墙策略）
