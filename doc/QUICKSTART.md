# 快速开始指南

## 🚀 5 分钟上手

### 步骤 1: 安装依赖 (2 分钟)

```bash
cd FileTransfer
pip install -r requirements.txt
```

**如果 aiortc 安装失败**，根据你的系统：

- **Windows**: 安装 [Visual Studio Build Tools](https://visualstudio.microsoft.com/downloads/)
- **Mac**: 运行 `xcode-select --install`
- **Linux**: `sudo apt install python3-dev libavformat-dev libavdevice-dev`

然后重新运行 `pip install aiortc`

### 步骤 2: 创建测试文件 (10 秒)

```bash
echo "这是一个测试文件" > test.txt
```

### 步骤 3: 开始传输 (3 分钟)

#### 在接收端机器上：

```bash
python receiver.py
```

你会看到：
```
============================================================
等待发送端的连接信息...
============================================================

请输入发送端提供的信息:
```

#### 在发送端机器上：

```bash
python sender.py test.txt
```

你会看到一长串字符，例如：
```
============================================================
请将以下信息发送给接收端（可以通过微信、QQ 等方式）
============================================================
H4sIAAAAAAAA/6VWTY/bNhC9+1cQvgiQZVmyP...
============================================================

请输入接收端返回的信息:
```

#### 交换信息：

1. 复制发送端的字符串
2. 粘贴到接收端，按回车
3. 接收端会生成响应字符串
4. 复制接收端的字符串
5. 粘贴到发送端，按回车

#### 观察传输：

```
✓ 连接已建立，开始传输...
文件名: test.txt
大小: 27.00 B
SHA256: a3b5c8d1e2f3...

发送进度: 27B/27B [████████████████████] 100%
✓ 文件传输完成！
```

### 🎉 完成！

接收端会显示：
```
✓ 文件接收完成且校验通过！
✓ 文件已保存到: ./received_test.txt
```

## 📱 实际使用场景

### 场景：给朋友发送一个 5GB 的视频

1. **朋友的电脑**（接收端）
   ```bash
   python receiver.py --output ~/Downloads
   ```

2. **你的电脑**（发送端）
   ```bash
   python sender.py ~/Movies/video.mp4
   ```

3. **通过微信/QQ 交换两次连接信息**

4. **完成！** 视频直接 P2P 传输，不经过任何云服务器

## ❓ 常见问题

### Q: 两台机器必须在同一局域网吗？

**A**: 不需要！这就是这个工具的核心优势。通过 NAT 穿透，可以跨网络传输。

### Q: 连接一直显示 "connecting"？

**A**:
1. 检查防火墙是否阻止了程序
2. 确保双方都能访问互联网
3. 如果还是失败，使用 ZeroTier/Tailscale 创建虚拟局域网

### Q: 传输速度如何？

**A**: 取决于你的上行带宽。通常能达到理论上限的 80-90%。

### Q: 支持断点续传吗？

**A**: 当前版本不支持。建议传输前确保网络稳定。

### Q: 安全吗？

**A**:
- ✅ 文件数据完全 P2P 传输
- ✅ WebRTC 默认加密
- ✅ SHA256 完整性校验
- ✅ 不经过任何第三方服务器
